# 📝 C群：イベント管理機能
## イベントとは
イベントは、クラブで開催され、クラブメンバーが参加する練習会、レッスン会、大会を指す。
### イベント構成要素
- イベントタイトル：イベントを識別する名前。「練習会」など
- 開催日：イベントが開催される日
- 開催開始時間：イベント開始時間
- 開催終了時間：イベント終了時間
- 場所：イベントが行われる場所
- 定員：イベントに参加可能な人数
- 最低催行人数：イベントを開催するために最低限必要な人数
- 参加レベル下限：イベントに参加可能なレベル下限値(DUPR値)
- 参加レベル上限：イベントに参加可能なレベル上限値(DUPR値)
- 駐車場可否：イベント会場の駐車場の有無
- 駐車場台数：イベント会場で使える駐車場数
- 試合形式：シングルス、ダブルス
- イベント形式：男子ダブルス、女子ダブルス、ミックスダブルスなど試合形式
- 単独参加可否：1人でも参加可能か、ペアでの参加のみか
- ゲスト参加可否：ゲストが参加できるか
- 費用：イベント参加に必要な費用
- ゲスト費用：ゲストが参加する場合の費用
- 申し込み開始日：イベント申込開始日
- 申し込み締め切り日：イベント申込終了日
- 抽選日：定員オーバー時の参加者を決定する抽選日
- 注記：イベントに関する注意事項など
- イベント状態：下書き、公開、無効
- イベント状況：募集前、募集中、応募締め切り後、参加者確定済み、中止、終了
- イベント作成者：イベントを作成した人
- イベント更新者：イベントを更新した人
- イベント作成日時：イベントを作成した日時
- イベント更新日時：イベントを更新した日時

### 参加者に関連する人の要素
- イベント登録者
- イベント参加登録日時
- イベント参加情報更新日時
- イベント参加ペア名(member_number)

## ユースケース
### 一般参加者
- C-01 イベント一覧参照：一般利用者は受付または公開中のイベントの概要をリストまたはカレンダー形式でイベント一覧を確認する。(イベント概要には、イベント名、日付、時間、場所、参加レベル、参加希望者が表示される)
  - C-01a：一般利用者は、終了したイベントの概要をリストまたはカレンダーから確認する
- C-02 イベント詳細参照：一般利用者は、イベントの日時、場所、費用、参加レベル、駐車場枠数、参加者、参加条件、注意事項を参照する。また、申し込み状況を確認する。
  - C-02a：一般利用者は、過去のイベントの詳細を参照する。
- C-03 参加申請：一般利用者は、イベントに参加申請する。参加時に駐車場希望の有無を選択できる。
- C-04 参加申請内容の変更：一般利用者は参加申請したイベントの募集期限内に、ペア情報や駐車場希望を書き換える。
  - C-04a：ペアが不参加になった時、ペア相手は応募期間外でも別のペアを指定できる。(当選権限の継承)
- C-05 参加申請の取り消し：一般利用者は参加申請したイベントをキャンセルする。ペア申込の場合は、ペアの申し込みは受付保留となる
- C-06 イベント管理者への連絡メモを書き込む
### イベント管理者
- C-11 イベント新規作成：イベント管理者は、イベントの日時、場所、定員、料金、レベル、駐車場枠、参加条件を新規登録する。
- C-12 イベント編集：イベント管理者は、作成済みのイベント内容を修正する。(抽選が終わったイベントに関しては、編集が制限される項目がある)
  - C-12a イベント公開ステータス変更：イベント管理者は、下書きから公開、または中止（非公開）へ状態を変更する。
- C-13 自動抽選実行：システムは、イベントに設定した期日に、優先ルール（役員・プレミア優先など）に基づき、当選者(参加順位)を決定する。参加順位が定員以内を当選者、定員を超えた場合はキャンセル待ちに振り分ける
  - C-13a 駐車場割り当て：駐車枠の当選者を最終決定する。
- C-14 手動抽選（個別当選操作）：イベント管理者は、任意の会員を当選・落選・キャンセル、キャンセル待ちへ振り分ける。
  - C-14a：ペアを差し替え。イベント管理者は、ペア情報の削除や差し替えを行う
- C-15 当日出席チェック：現場で実際の出席・欠席、参加費支払いを記録する。
- C-16 請求額確定・会計連携：出席実績に基づき、キャンセル料を含む最終額を確定し、会計群へ送信する。
- C-17 参加者への連絡メモを書き込む
